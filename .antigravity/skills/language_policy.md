# Language Policy

## Code Language
All programming-related entities must be named in **English**. This includes:
- Variable names
- Function names
- Classes and Interfaces
- File names
- Database tables and columns
- Commit messages (as per standard practice, though the user might prefer Spanish, I'll stick to English for code entities)

## Interface Language
All user-facing text must be in **Spanish**. This includes:
- Button labels
- Page titles
- Navigation items
- Success/Error messages
- Form placeholders
- Tooltips and captions
- Any text rendered in the browser for the end-user.

## Rationale
Maintains a professional, globally standard codebase while providing a localized, native experience for the target audience of Literudo.
